🔐 Cofre Eletrônico Nios II (FPGA Altera DE2)Este repositório contém a implementação completa, código-fonte e materiais de apresentação de um sistema de controle de acesso (Cofre Eletrônico). O projeto foi desenvolvido utilizando um processador soft-core Nios II embarcado em uma FPGA Altera/Intel Cyclone II (Placa DE2).O projeto destaca o conceito de Co-design de Hardware e Software, integrando a síntese de uma arquitetura customizada via Qsys/Quartus com a programação de baixo nível (bare-metal) em C.📁 Estrutura do RepositórioNeste repositório, você encontrará os seguintes arquivos:📄 codigo_main.txt: O código-fonte em C (bare-metal) contendo toda a lógica do cofre, o driver customizado do LCD e as chamadas de mapeamento de memória (IOWR/IORD).📦 Projeto_Quartus.qar: O arquivo de hardware (Quartus Archive). Contém a lógica de Top-Level em Verilog, o design do microcomputador no Qsys e os mapeamentos de pinos da placa DE2.🎥 Video_Implementacao.mp4 (ou link na descrição): Vídeo demonstrando o cofre funcionando na prática, incluindo a digitação da senha, acionamento dos LEDs, display LCD e o bloqueio de segurança após 3 tentativas falhas.📊 Apresentacao_Cofre.pdf (ou .pptx): Slides da apresentação defendida perante a banca avaliadora, detalhando a arquitetura e as soluções de engenharia.🛠️ Tecnologias e FerramentasHardware: Placa de Desenvolvimento Altera DE2 (Cyclone II EP2C35F672C6)Desenvolvimento de Hardware: Quartus II, Qsys (System Integration Tool)Linguagem HDL (Top-Level): VerilogSoftware IDE: Nios II Software Build Tools for EclipseLinguagem de Programação: C (Sem Sistema Operacional / Bare-metal)🧠 Arquitetura do Sistema (SOPC)O sistema foi desenhado para ser um computador customizado em um único chip (System on a Programmable Chip). O roteamento interno utiliza o Barramento Avalon (Avalon Switch Fabric) para comunicar o processador aos periféricos através de Memory-Mapped I/O.Componentes instanciados no Qsys:Nios II Processor Core: O "Cérebro" rodando a 50MHz.On-Chip Memory: Memória RAM/ROM do sistema.PIOs (Parallel I/O): Para leitura das Switches/Botões (senha) e acionamento dos LEDs de status.Avalon Character LCD Controller: Interface de comunicação com o display.System ID Peripheral e JTAG UART: Para segurança de sincronia Hardware/Software e depuração.💻 Desenvolvimento do Software (Bare-Metal)O código em C interage diretamente com o hardware físico, sem o auxílio de bibliotecas de alto nível prontas.Memory-Mapped I/O: Uso de macros IOWR e IORD para acessar endereços hexadecimais, realizando um Bypass na memória Cache do Nios II e garantindo comunicação instantânea com os pinos da placa.Driver de LCD Customizado: Controle por injeção de registradores (Offset 0 para comandos de tela e Offset 2 para caracteres ASCII). Implementação de rotinas de atraso manual (usleep) para sincronizar a alta frequência da CPU (50MHz) com a lentidão mecânica do hardware do LCD.🚀 Como FuncionaO sistema inicia solicitando a inserção da senha no display LCD.O usuário insere o código binário utilizando as Switches (SW) da placa DE2.A confirmação da tentativa é feita através do botão KEY.Acesso Permitido: O LED verde acende e o display exibe a mensagem de sucesso.Acesso Negado: O LED vermelho acende. Após 3 tentativas incorretas, o sistema entra em modo de bloqueio máximo, travando a CPU por 10 segundos por questões de segurança antes de permitir uma nova tentativa.Projeto desenvolvido para fins acadêmicos como demonstração prática de Arquitetura de Computadores e Sistemas Embarcados.
